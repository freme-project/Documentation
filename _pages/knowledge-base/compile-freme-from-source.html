This Guide will help you to recognise the requisites and the way you can start compiling FREME API.

Before getting started with FREME API you need to know that there are 3 Prerequisites you have to accomplish, these are: Java Runtime Edition and Java Development Kit (at least 7 of them); Maven and Git. You need to have all of them installed in your machine.

1st Step: Add maven repository credentials

for FREME API we have a private maven repository available, so the following step consists in providing a username and a password for your? repository.

1.1 XML File

To do it, you just need to create a xml file “settings.xml” with the following content: 


<settings xmlns="http://maven.apache.org/SETTINGS/1.0.0"
  xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
  xsi:schemaLocation="http://maven.apache.org/SETTINGS/1.0.0
                  http://maven.apache.org/xsd/settings-1.0.0.xsd">
  <servers>
    <server>
      <id>freme-nexus</id>
      <username>freme-nexus</username>
      <password>THE PASSWORD</password>
    </server>
  </servers>
</settings>

1.2 Location:

For machines using linux or mac: ~/.m2/settings.xml

For machines using windows: USERHOME/.m2/settings.xml

1.3 Password

Ask @jnehring for the password.

Now you can git clone any of the Git repositories. Maven will automatically download FREME dependencies from the maven repository. When you push a change to GitHub master branch then the Jenkins server will automatically create a new maven artifact and upload that artifact to Jenkins. So the maven repository always reflects the state of the master branches in the repositories.

2nd Step: Compile a Project

To compile any of the projects, use the mvn install command. All dependencies will be downloaded automatically. The FREME artifacts in the maven repository are automatically kept up to date by our continuous integration server. Upon every commit in a master branch it will upload the current SNAPSHOT version to the maven snapshot repository. During the release process the release artifacts are uploaded automatically to the maven release repository.

3rd Step: Create FREME distribution

This creates the FREME distribution in folder target/FREME-VERSION-full/FREME-VERSION/ where VERSION is the version number of FREME.

Execute the following script: 


git clone https://github.com/freme-project/Broker.git
cd Broker
mvn package

4th Steps: Failing unit tests

FREME relies on some external APIs and when these APIs are unavailable then some unit tests might fail.

When this happens then you use the skipTests=true option to run the maven goal without unit tests, e.g.:

mvn install -DskipTests=true.

5th Step: Start server

To start FREME on http://localhost:8080

For machines using Linux or Mac:

Open a command line in the distribution folder and type:


chmod +x bin/start_local.sh
bin/start_local.sh

For machines using Windows:

Open distribution folder in the following command line:


java -cp "./*:conf" eu.freme.broker.Broker

6th Action: Create execution environment in IDE

You can start the broker in your favourite IDE creating an execution environment that uses the following main class: eu.freme.broker.Broker
