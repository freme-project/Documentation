Initialising FREME

This article explains how to initialise the FREME database.

Prepare the database

The database connection is defined in the `application.properties` file, see FREME configuration options.

To add the default FREME entries to the tables, execute the following MySQL scripts at your FREME database.

  • Templates: add all FREME Templates
  • Sparql Converters (Filters): add all FREME sparql filters

Hibernate provides the possibility to autoload content into the database when the database schema is created. To autoload the FREME entities mentioned above, do the following:

  1. Put the templates script, the sparql converters script and this init-admin script into your FREME config folder. The init-admin script is necessary to meet the foreign key requirements of the other tables. Note that this overwrites `admin.username` and `admin.password` in your `application.properties` file.
  2. Change `CHANGE_THIS_PASSWORD` in the admin-script to a secure admin password.
  3. Add the following entries to your `application.properties` file: 
    spring.jpa.properties.hibernate.hbm2ddl.import_files=init-admin.sql, init-sparqlconverter.sql, init-template.sql
spring.jpa.properties.hibernate.hbm2ddl.import_files_sql_extractor=org.hibernate.tool.hbm2ddl.MultipleLinesSqlCommandExtractor
spring.jpa.hibernate.ddl-auto=create
  4. Start FREME
  5. Remove the added entries from your `application.properties` file to prevent hibernate from re-creating the databases (and dropping its previous content) when you restart it.
