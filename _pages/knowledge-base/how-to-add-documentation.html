Here you can find out how to add more pages to this website

This website is built using Jekyll, which makes it very easy to add new content.

Start by cloning the Documentation Repository.

In the directory _pages you will see different files and folders.

Depending on where you want your content to be found in the navigation bar, add a new file. This file can be Markdown or HTML.

The front matter YAML

For an in-depth explanation on what is possible with front matter YAML, see the Jekyll Front Matter YAML Documentation

Here we will only go over the basics:

Every file that should be displayed on this time needs to have front matter YAML, enclosed by a set of triple-dashed lines

Example: This Page

---
layout: page
title: Adding Documentation
header: How to add Documentation
dropdown: Knowledge Base
pos: 4.5
---
  • Layout:

    In the directoy _layouts you can find defined layouts for various types of content. As of now there is one for a basic page like this one, and one for integrating a swagger-UI like this one

  • Title:

    This determines what this page will be called in the navigation bar.

  • Header:

    This determines the title at the top of this page when it is the active page. When left blank, the Title will be used

  • Dropdown:

    Specifies the name of the Dropdown menu where the content will be located. If left blank, it is added in a new column.

  • Pos:

    The position at which the content will be placed (smallest pos is the most left item). Dropdowns can be ordered by adding a decimal.

The content

Now comes your content!

Any valid Markdown will be rendered correctly.

If you choose to use HTML instead, you do not have to include a head, header, or footer, you can get started with your content straight away!
