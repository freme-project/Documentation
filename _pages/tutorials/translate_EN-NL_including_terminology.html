How to translate from English to Dutch with e-Translation and e-Terminology

Introduction

Tutorial demonstrates usage of e-Translation and using more features to improve translation quality.

Use e-Terminology before e-Translation to improve or adjust translated output. e-Terminology identifies terms within the source text and helps machine translation to use them. You can introduce your own terminology collection to force MT using your own terminology from term.tilde.com portal.

Technical Instructions

Step 1 - using e-Terminology

Annotate terms within the source text.

cURL

curl -X POST -d "The EU in brief. The EU is a unique economic and political partnership between 28 European countries that together cover much of the continent." "{{site.apiurl | prepend: site.url}}/e-terminology/tilde?informat=text&outformat=json-ld&source-lang=en&target-lang=nl"

Result

There are following terms found (term rank in brackets): continent [1], partnership [0.8961], briefing [0.5946], economics [0.3165], covering [0]

Step 2 - using e-Translation to translate

Translate text taking in account annotated term in Step 1.

The input data is the output of Step 1

cURL

curl -X POST -d "response-of-step1.ttl" "{{site.apiurl | prepend: site.url}}/e-translation/tilde?informat=json-ld&outformat=json-ld&source-lang=en&target-lang=nl";

Where the file response-of-step1.ttl is the response of e-Terminology.

Result

Result of machine translation is: De voorschriften in DE EU. De EU is een uniek partnerschap tussen politiek en economie in de Europese landen, die gezamenlijk 28 verpakking van het continent.
With MT confidence score 0.66955627717300892

Try out API Calls

For demonstration purposes we included an API call to /e-terminology/tilde and /e-translation/tilde. This tutorial covers only the most important parameters of this API endpoint. You can find an extensive overview on this API call in the complete API documentation.

Request

Query string parameter input
Specify the text you want to translate here.


Query string parameter source-lang
Enter the language you want to translate from


Query string parameter source-lang
Enter the language you want to translate to


Query string parameter informat
Specifies the format of the input data. In this example we submit plaintext so we set this parameter to "text"


Query string parameter outformat
Specifies the format of the data returned by FREME. In this example we want to return JSON-LD data that can be easily processed by many programming languages so we specify "outformat=json-ld"


Response

Response of Step 1: e-Terminology 

Here you will get the intermediate results from e-Terminology, that will be submitted to e-Translation.

Response of Step 2: e-Translation

Here you will get the results from e-Translation.

Using Pipeline - alternative approach

Query e-services with single API call

Instead of querying the FREME server in two steps, you can also use pipeline to force chaining the services on server side. Follow the pipelining tutorial for more information.

cURL

curl -X POST -H "Content-Type: application/json" -d "[
{
\"method\": \"POST\",
\"endpoint\": \"{{site.apiurl | prepend: site.url}}/e-terminology/tilde\",
\"parameters\": {
\"source-lang\": \"en\",
\"target-lang\": \"nl\"
},
\"headers\": {
\"content-type\": \"text/plain\",
\"accept\": \"application/json+ld\"
},
\"body\": \"The EU in brief. The EU is a unique economic and political partnership between 28 European countries that together cover much of the continent.\"
},
{
\"method\": \"POST\",
\"endpoint\": \"{{site.apiurl | prepend: site.url}}/e-translation/tilde\",
\"parameters\": {
\"source-lang\": \"en\",
\"target-lang\": \"nl\"
},
\"headers\": {
\"content-type\": \"application/json+ld\",
\"accept\": \"application/json+ld\"
}
}
]
" "{{site.apiurl | prepend: site.url}}/pipelining/chain"

Result

Results from both services are in the output (same as in Step 2 - using e-Translation to translate):

There following terms are found (term rank in brackets): continent [1], partnership [0.8961], briefing [0.5946], economics [0.3165], covering [0]

Result of machine translation is: De voorschriften in DE EU. De EU is een uniek partnerschap tussen politiek en economie in de Europese landen, die gezamenlijk 28 verpakking van het continent.
With MT confidence score 0.66955627717300892

Use of SPARQL filters

Sometimes there is no need for complex RDF data structures. If so, then you can retrieve also a predefinied structure of data using SPARQL filtering possibilities of the FREME server. See more about thes feature here this article

SPARQL filters can be used together with pipelines. Firstly must find existing filter or create new one, and then by adding the filter name as parameter to the pipeline configuration.

Customising e-Translation

Selecting translation system

Translation system can be selected by setting the source language and the target language for each request. It is possible that there is more translation systems for the same language pair, but in different domain. So it is possible to set optionaly also domain parameter:

HTML Request

POST /current/e-translation/tilde?source-lang=en&target-lang=de&domain=legal HTTP/1.1
            Host: api.freme-project.eu
            Accept: application/rdf+xml
            Content-Type: text/plain
            The EU in brief. The EU is a unique economic and political partnership between 28 European countries that together cover much of the continent.

Curl example

curl -X POST -H "Accept: application/rdf+xml" -H "Content-Type: text/plain" -d 'The EU in brief. The EU is a unique economic and political partnership between 28 European countries that together cover much of the continent.' "http://api.freme-project.eu/current/e-translation/tilde?source-lang=en&target-lang=de&domain=legal"

In case of non-existing domain and language pair the server will respond with HTTP status code 404 Not Found

ITS 2.0 category support

e-Translation supports ITS 2.0 data categories. See complete list of the supported categories and usage samples in the article e-Internationalization

ITS 2.0 data categories provides possibility to include MT confidence score to the output of e-Translation. This allows the post editors or other automatic or manual evalueator easier spot the ares with potentiotially weak translation.

Terminology markup and additional terminology information give possibility to tune the results of e-Translation. ITS 2.0 terminology supporting categories allows to create a bridge between two independent e-Services - supports transfering information from e-Terminology to e-Translation. This is done by annotation source text with NIF objects -terms, and additionaly provideing terminology information in OntoLex Lemon. MT engine extracts term translation in target language from OntoLex data structures and chooses the best translation based on terminology provided.

Also category translate is widelly used and helps improve translation results by pointing out the areas of the source document that is not intended for translation.

List of systems

See this list of available public demo systems here: List of translation systems

There are following statuses of systems available

:
  • Running - ready for translation;
  • Waking up - will be ready for translation very soon
  • Stand-by - systems in stand-by mode
  • Failed - systems with error. Normally such section not visible.

Available systems mean that they are ready for translation and standby systems means that need to awake. Systems will be awaked with the first request and needs some minutes to start up. It will go for sleep if inactive 60 minutes (no translations). Some systems are always in runing mode, if you find that this stand-by mode is problem, contact Tilde to resolve it.

Private systems

It is possible to train translation system on private and propriaty data, and then use it through the FREME server, but still keep it secure and confidential. Private systems are not listed on public system list and will be accessible by providing a private key. That private key can be retrieved only by contacting Tilde.

to use private MT system, add your key to the query string parameter list:

HTML Request

POST /current/e-translation/tilde?source-lang=en&target-lang=de&key=u-iuo-2323-444-223-ABC HTTP/1.1
Host: api.freme-project.eu
Accept: application/rdf+xml
Content-Type: text/plain

The EU in brief. The EU is a unique economic and political partnership between 28 European countries that together cover much of the continent.

In case of incorect key the server will return HTTP status code 401 Unauthorized.
